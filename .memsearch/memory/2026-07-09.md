- User reported that pasting anything into cmux crashes the app.
- Claude Code reproduced the crash (`free(): double free detected in tcache 2`) on the patched debug build using multi-line clipboard text, then captured a GDB backtrace showing the abort in `Surface.startClipboardRequest()` during keyboard event handling.
- Investigated clipboard callback contract by examining ghostty.h header typedefs, bindgen-generated FFI bindings, and ghostty's embedded.zig reference implementation; identified two compounding bugs: (1) patched root `ghostty.h` incorrectly declared `read_clipboard_cb` returning `void` instead of `bool`, causing ghostty to read garbage "not started" value, and (2) the Rust callback completed requests synchronously inline (via `block_on`), freeing the request state while ghostty also freed it.
- Applied fixes: corrected `ghostty.h` return type to `bool` (triggering bindgen regeneration), rewrote `read_clipboard_cb` to return `bool` immediately and complete asynchronously (matching ghostty's contract), and fixed latent arg mismatch in `confirm_read_clipboard_cb`.
- Rebuilt debug binary and verified multi-line paste ×2 and single-token paste all survive and text lands correctly in the terminal; noted the running AppImage (v0.1.0) is still the unfixed upstream version.
